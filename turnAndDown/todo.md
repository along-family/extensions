# Chrome 扩展需求：翻页懒加载预加载器

## 1. 项目背景

用户经常访问一个分页列表类型的网页，URL 中包含 `page=数字` 参数（例如 `https://example.com/list?page=10`）。

该网页实现了懒加载机制：页面初始化时只加载可视区域的内容，用户必须**滚动页面到底部**，然后等待 **5-10 秒**，新的图片和文字才会加载出来。如果继续往下翻，需要反复滚动和等待。

用户希望一次性阅读多个分页的内容（例如第 10 页到第 6 页），但手动操作非常痛苦：需要打开多个 tab、每个 tab 手动滚动、每个 tab 等待数秒。

## 2. 核心需求

开发一个 Chrome 浏览器扩展，实现以下功能：

**输入：**
- 一个带 `page=数字` 参数的 URL（代表当前页）
- 一个整数 N（表示向前翻页的数量，即同时加载当前页及之前 N-1 页）

**输出：**
- 自动在后台依次打开 N 个 tab，页号分别为：当前页、当前页-1、当前页-2 ... 直到当前页-N+1
- 每个 tab 打开后，自动滚动到页面底部
- 滚动后自动等待一段时间（可配置，默认 5-10 秒），触发该页面的懒加载
- 全部 N 个 tab 处理完成后，通知用户

**关键约束：**
- 整个预加载过程必须在**后台**运行。用户启动任务后，可以最小化浏览器、切换到其他应用（如看视频、写文档），预加载不应中断
- 预加载过程中不应抢夺焦点或自动激活 tab 干扰用户
- 预加载完成后，所有 tab 保留在浏览器中，用户可随时切换阅读

## 3. 功能细节

### 3.1 URL 解析与生成
- 从输入的 URL 中解析出 `page` 参数的值（支持 `?page=数字` 或 `&page=数字` 格式）
- 根据当前 page 值 N，生成 N 个 URL：`page = 当前`, `当前-1`, `当前-2`, ... 直到 `当前-N+1`
- 如果 page 数字小于 1，则停止生成（不生成 page=0 或负数的 URL）
- 保留原 URL 中的所有其他参数和路径，仅替换 `page` 的值

### 3.2 Tab 管理
- 使用 `chrome.tabs.create` API 创建新 tab，必须设置 `active: false`，确保 tab 在后台创建，不干扰用户当前操作
- 按顺序处理：等待前一个 tab 完成加载和等待后，再创建下一个 tab（避免网络并发拥堵）
- 预加载完成后，所有 tab 保持打开状态，不自动关闭

### 3.3 页面加载触发
- 每个 tab 创建后，需等待页面 `status === "complete"` 再执行下一步
- 然后向该 tab 注入脚本，执行 `window.scrollTo(0, document.body.scrollHeight)` 滚动到底部
- 为了更可靠地触发懒加载，可考虑执行两次滚动（间隔约 500ms）
- 滚动后，等待一个可配置的延迟时间（默认 8 秒），让图片和内容完成加载

### 3.4 用户界面（Popup）
- 扩展图标点击后弹出一个小窗口，包含：
  - URL 输入框（预填示例或上次使用的 URL）
  - 翻页数量输入框（数字，默认 5，范围 1-20）
  - 每个页面等待时间输入框（数字，单位秒，默认 8，范围 3-30）
  - 开始按钮
  - 状态显示区域（显示当前进度、成功/失败信息）
- 界面简洁，操作直观

### 3.5 后台运行
- 使用 Manifest V3 的 Service Worker（`background.js`）处理实际预加载逻辑
- Popup 点击开始后，将 URL 列表和配置参数发送给 Service Worker
- Service Worker 收到消息后立即开始处理，即使用户关闭 Popup 也不影响

### 3.6 用户反馈与通知
- 预加载开始时，Popup 状态区域显示“已启动，后台运行中”
- 预加载完成后，Service Worker 通过 `chrome.notifications` API 发送系统通知（如果权限允许），告知用户已完成及成功加载的页面数量
- 每个页面的处理结果（成功/失败）应输出到浏览器控制台，便于调试

## 4. 非功能性需求

- **兼容性**：仅需支持最新版 Chrome（Manifest V3）
- **稳定性**：单个页面加载失败不应影响其他页面的预加载（捕获异常，继续执行）
- **超时保护**：每个页面的加载等待设置超时（如 30 秒），避免卡死
- **权限最小化**：仅请求必要的权限（`tabs`, `scripting`, `notifications`，以及 `<all_urls>` 或目标网站的 host permission）

## 5. 使用示例

用户当前在浏览 `https://news.com/articles?page=10&category=tech`，希望提前加载第 10、9、8、7、6 页。

操作步骤：
1. 点击扩展图标
2. URL 输入框中填入上述 URL（或自动检测当前页面 URL）
3. 翻页数量填入 5
4. 等待时间保持 8 秒
5. 点击“开始”
6. 用户最小化浏览器，去看视频
7. 约 40 秒后收到系统通知：“预加载完成，已加载 5 个页面”
8. 用户回到浏览器，依次点击第 6-10 页的 tab，所有内容已加载完成，无需等待

## 6. 可选增强（非必需，可列出供后续迭代）

- 自动检测当前活动标签页的 URL，预填到输入框
- 支持“向后翻页”（page++）的选项
- 支持对同一网站的滚动等待时间记忆
- 支持暂停/取消正在进行的预加载任务
- 将加载完成的 tab 自动分组

## 7. 验收标准

1. 输入包含 `page=数字` 的 URL 和 N 值后，能正确生成 N 个 URL，page 值递减且不小于 1
2. N 个 tab 在后台依次打开，不抢夺焦点
3. 每个 tab 自动滚动到底部
4. 每个 tab 滚动后等待指定秒数
5. 用户在此期间可以切换浏览器标签页或切换到其他应用，预加载不中断
6. 全部完成后发送系统通知
7. 所有 tab 内容已加载（用户手动滚动时无等待或等待时间明显缩短）
